How DigiCert Services API Works

DigiCert Services API utilizes the REST API simple architectural style and the existing technology and protocols of HTTP.

REST libraries are included in all major programming languages

Return JSON or XML responses

Supports UTF-8 character set encoding for all data

Uses standard HTTP method calls (available on every language and platform) to make requests and retrieve information

DigiCert API Access

Your DigiCert account representative will need to activate the DigiCert Services API within your CertCentral account. From there, you can generate API keys from your Profile Settings page. Your Account ID functions as the username for API access.

API Features and Benefits

Streamlined Certificate Management

  • Configure product settings
  • Make requests, request order status, renew certificates
  • Revoke, approve or reject requests, reissue certificates
  • Download certificates in different formats
  • Receive certificate renewal notifications (non-escalation and escalation)
  • Run scans and view reports

Account Management

  • Configure email notifications
  • Manage password requirements
  • Set up two-factor authentication
  • Configure IP access restrictions
  • Run audits/create audit log notifications

Management for Domains and Organizations

  • Add and authorize organizations for certificates
  • Add and authorize domains for certificates
  • Enable DNS CNAME domain control validation (DCV)

Management for Individuals and Teams

  • Add users
  • Create and Revoke API keys
  • Create guest URLs
  • Add divisions
  • Create division Ekeys
